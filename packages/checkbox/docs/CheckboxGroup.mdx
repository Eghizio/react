# CheckboxGroup

Group [Checkboxes](/Checkbox) together under a common label and help text.

<PackageInfo
    packageData={packageData}
    componentNames={['Checkbox', 'CheckboxGroup']}
/>

## Example

```jsx example
<CheckboxGroup label="Favorite colors">
    <Checkbox value="r">Red</Checkbox>
    <Checkbox value="g">Green</Checkbox>
    <Checkbox value="b">Blue</Checkbox>
</CheckboxGroup>
```

## Migrating from Troika

-   You should no longer include or import any Troika checkbox CSS.
-   **BREAKING**:`onChange` now emits a an array of string values instead of a
    change event as its first parameter.
-   `error` has been deprecated. Use `invalid` instead.

## Content

A CheckboxGroup accepts multiple [Checkboxes](/Checkbox) as children.

**Note**: The [Checkboxes](/Checkbox) must have their `value` prop set when
used inside a CheckboxGroup.

## Value

CheckboxGroup supports selecting zero or more items. An initial, uncontrolled
value can be provided to the CheckboxGroup using the `defaultValue` prop.
Alternatively, a controlled value can be provided using the value prop. Both of
these props accept an array of selected items, which map to the value prop on
each item.

```jsx example
function Example() {
    let [checked, setChecked] = React.useState(['g', 'b']);

    return (
        <div className="flex space-x-48">
            <CheckboxGroup
                label="Favorite colors (uncontrolled)"
                defaultValue={['g', 'b']}
            >
                <Checkbox value="r">Red</Checkbox>
                <Checkbox value="g">Green</Checkbox>
                <Checkbox value="b">Blue</Checkbox>
            </CheckboxGroup>

            <CheckboxGroup
                label="Favorite colors (controlled)"
                value={checked}
                onChange={setChecked}
            >
                <Checkbox value="r">Red</Checkbox>
                <Checkbox value="g">Green</Checkbox>
                <Checkbox value="b">Blue</Checkbox>
            </CheckboxGroup>
        </div>
    );
}
```

## Labeling

A visual label should be provided for the CheckboxGroup using the `label` prop.

### Accessibility

If a visible label isn't specified for the CheckboxGroup, an `aria-label` must
be provided for accessibility. If the field is labeled by a separate element, an
`aria-labelledby` prop must be provided using the id of the labeling element
instead.

## Help text

CheckboxGroups can provide additional context with `helpText` if the label and
checkboxes aren't enough.

```jsx example
<CheckboxGroup
    label="Favorite colors"
    helpText="Skip this if you're color blind"
>
    <Checkbox value="r">Red</Checkbox>
    <Checkbox value="g">Green</Checkbox>
    <Checkbox value="b">Blue</Checkbox>
</CheckboxGroup>
```

## Validation

CheckboxGroups can communicate to the user whether the current value is invalid.
Implement your own validation logic in your app and set the `error` prop to
display it as invalid.

`error` is often paired with `helpText` to provide feedback to the user about
the error.

```jsx example
<CheckboxGroup
    label="Favorite colors"
    invalid
    helpText="Green surely isn't amongst your favorites?"
    defaultValue={['g']}
>
    <Checkbox value="r">Red</Checkbox>
    <Checkbox value="g">Green</Checkbox>
    <Checkbox value="b">Blue</Checkbox>
</CheckboxGroup>
```
